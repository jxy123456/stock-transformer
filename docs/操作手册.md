## 2. 数据下载

### 2.1 下载全 A 股数据

```bash
python scripts/download_data.py --all --start 20100101
```

这会执行以下步骤：

1. 调用 akshare 获取全 A 股股票列表（约 5000 只）
2. 过滤掉北交所(8xxxxx)和新三板(4xxxxx)，保留沪深主板(60/00) + 创业板(300) + 科创板(688)
3. 逐只下载三类数据：
   - **日线行情**: OHLCV + 换手率（前复权）
   - **估值指标**: PE/PB/PS/股息率（日频）
   - **财报摘要**: ROE/营收增速/净利增速等（季频，forward-fill 到日频）
4. 计算技术特征 + 基本面特征，保存为 CSV

下载过程中每只股票间隔 0.5 秒，避免被限流。失败的股票代码写入 `outputs/logs/failed_{type}.txt`。

### 2.2 下载指定股票

```bash
python scripts/download_data.py --symbols 000001 600519 000858 --start 20150101
```

常用股票代码：

| 代码 | 名称 | 板块 |
|------|------|------|
| 000001 | 平安银行 | 深主板 |
| 000858 | 五粮液 | 深主板 |
| 600519 | 贵州茅台 | 沪主板 |
| 300750 | 宁德时代 | 创业板 |
| 688981 | 中芯国际 | 科创板 |

### 2.3 只下载行情或只下载财报

```bash
# 只下载行情（跳过财报和估值）
python scripts/download_data.py --all --no-fundamental

# 只下载财报（跳过行情）
python scripts/download_data.py --all --no-market

# 跳过特征计算
python scripts/download_data.py --all --no-features
```

### 2.4 断点续传

如果下载中途中断，用 `--skip-cached` 跳过已有缓存继续：

```bash

```

## 3. 数据增量更新

### 3.1 每日更新

```bash
# 更新全 A 股（只拉最后日期之后的新数据）
python scripts/update_data.py --all

# 更新指定股票
python scripts/update_data.py --symbols 000001 600519
```

增量逻辑：
- **行情**: 读取本地 Parquet 最后一条日期，从 `last_date+1` 拉到今天，追加并按日期去重
- **估值/财报**: 数据量小，直接全量覆盖
- **特征**: 对有更新的股票重新计算

### 3.2 强制全量重下

```bash
python scripts/update_data.py --all --force
```

### 3.3 建议的定时任务

每个交易日收盘后（15:05）运行增量更新：

```bash
# crontab 示例
5 15 * * 1-5 cd /path/to/stock && python scripts/update_data.py --all
```

## 4. 模型训练

### 4.1 训练单只股票

```bash
python scripts/train.py --symbol 000001 --start 20200101 --end 20241231
```

### 4.2 指定 GPU

```bash
python scripts/train.py --symbol 000001 --device cuda
```

### 4.3 训练流程

脚本内部执行以下步骤：

1. 加载该股票的行情数据，计算技术特征
2. 生成 Walk-Forward 滑动窗口：
   - 训练窗口 500 天，验证 100 天，测试 50 天，步长 50 天
   - 每个 fold 独立计算标准化参数（只在训练窗口上 fit，杜绝未来信息泄露）
3. 对每个 fold：
   - 创建全新的 StockTransformer 模型
   - 训练最多 100 个 epoch，早停 patience=15
   - 在测试集上评估
4. 输出所有 fold 的平均指标

### 4.4 训练输出

```
Fold 0 test: dir_acc=0.542 IC=0.0312 RMSE=0.015234
Fold 1 test: dir_acc=0.558 IC=0.0287 RMSE=0.014891
...
======================================== Summary ========================================
Avg directional accuracy: 0.550
Avg IC: 0.0298
Avg Rank IC: 0.0315
Avg RMSE: 0.015062
```

方向准确率 > 50% 即有交易价值。IC > 0.03 在 A 股属于可接受水平。

### 4.5 模型检查点

保存在 `outputs/checkpoints/`：

```

000001_fold0_20250515.pt  # fold 0 时间戳检查点
000001_fold1_best.pt      # fold 1 最佳模型
```

## 5. 回测

### 5.1 运行回测

```bash

```

### 5.2 自定义初始资金

```bash
python scripts/backtest.py --symbols 000001 --cash 500000
```

### 5.3 回测规则

回测严格遵循 A 股交易规则：

| 规则 | 实现 |
|------|------|
| T+1 | 当日买入不可当日卖出 |
| 涨跌停 | 主板 ±10%，创业板/科创板 ±20%，超限价订单拒绝 |
| 印花税 | 0.1%，仅卖出时收取 |
| 佣金 | 0.03% 双边，最低 5 元/笔 |
| 滑点 | 5 个基点 |
| 手数 | 100 股整数倍 |
| 执行时序 | T 日收盘产生信号 → T+1 日开盘执行 |

## 8. 配置说明

所有参数在 `config/default.yaml` 中定义。修改配置后无需改代码，直接生效。

### 8.1 数据配置

```yaml
data:
  source: "akshare"              # 数据源: akshare 或 tushare
  tushare_token: ""              # tushare API token（使用 tushare 时必填）
  symbols: ["000001", "600519", "000858"]  # 默认股票池
  start_date: "20200101"         # 默认起始日期
  end_date: "20241231"           # 默认结束日期
  adjust: "qfq"                  # 复权方式: qfq=前复权, hfq=后复权, ""=不复权
  cache_dir: "outputs/data_cache"
  cache_expiry_hours: 24
```

### 8.2 特征配置

```yaml
features:
  ma_periods: [5, 10, 20, 60]   # SMA 周期
  ema_periods: [12, 26]         # EMA 周期
  macd_fast: 12                  # MACD 快线
  macd_slow: 26                  # MACD 慢线
  macd_signal: 9                 # MACD 信号线
  rsi_period: 14                 # RSI 周期
  bollinger_period: 20           # 布林带周期
  bollinger_std: 2.0             # 布林带标准差倍数
  atr_period: 14                 # ATR 周期
  volume_ma_period: 20           # 成交量均线周期
```

### 8.3 模型配置

```yaml
model:
  d_model: 128                   # 隐藏维度
  n_heads: 8                     # 注意力头数
  n_layers: 4                    # Encoder 层数
  d_ff: 512                      # FFN 中间维度
  dropout: 0.1                   # Dropout 率
  seq_len: 60                    # 输入序列长度（交易日）
  pred_len: 5                    # 预测天数
  pos_encoding: "sinusoidal"     # 位置编码: sinusoidal 或 learned
  pooling: "attention"           # 池化方式: attention 或 mean
```

### 8.4 训练配置

```yaml
training:
  lr: 0.0005                     # 学习率
  weight_decay: 0.01             # L2 正则化
  batch_size: 64
  epochs: 100                    # 最大训练轮数
  early_stopping_patience: 15    # 早停耐心值
  gradient_clip_norm: 1.0        # 梯度裁剪
  loss: "huber"                  # 损失函数: huber / mse / mae
  huber_delta: 1.0               # Huber 损失 delta
  scheduler: "cosine_warmup"     # 学习率调度
  warmup_epochs: 5               # 预热轮数
  walk_forward:
    train_window: 500            # 训练窗口（天）
    val_window: 100              # 验证窗口
    test_window: 50              # 测试窗口
    step: 50                     # 滑动步长
  normalization: "zscore"        # 标准化: zscore / minmax
  normalize_per_fold: true       # 每 fold 重新计算标准化参数
```

### 8.5 回测配置

```yaml
backtest:
  initial_cash: 1000000          # 初始资金（元）
  commission_rate: 0.0003        # 佣金费率 0.03%
  stamp_tax_rate: 0.001          # 印花税率 0.1%（仅卖出）
  min_commission: 5.0            # 最低佣金 5 元
  slippage_bps: 5                # 滑点 5 基点
  benchmark: "000300"            # 基准指数（沪深300）
  risk_free_rate: 0.02           # 无风险利率（Sharpe 计算）
```

### 8.6 策略配置

```yaml
strategy:
  buy_threshold: 0.005           # 买入阈值：预测收益率 > 0.5% 买入
  sell_threshold: -0.005         # 卖出阈值：预测收益率 < -0.5% 卖出
  sizing_method: "vol_adjusted"  # 仓位方法: vol_adjusted / fixed_fraction / kelly
  max_position_pct: 0.20         # 单只最大仓位 20%
```

### 8.7 风控配置

```yaml
risk:
  max_positions: 5               # 最多持有 5 只
  max_position_pct: 0.25         # 单只仓位上限 25%
  max_drawdown_pct: 0.15         # 熔断回撤阈值 15%
  reduce_at_pct: 0.10            # 减仓回撤阈值 10%
  stop_loss_pct: 0.08            # 止损 -8%
  take_profit_pct: 0.15          # 止盈 +15%
  trailing_stop_pct: 0.05        # 追踪止损 5%
  max_sector_pct: 0.40           # 行业集中度上限 40%
```

### 8.8 配置覆盖

在 `config/` 目录下新建 YAML 文件即可覆盖默认值（深合并），例如 `config/local.yaml`：

## 10. 模型架构

```
Input (batch, 60, num_features)
  │
  ▼  Linear(num_features → 128)
  │
  ▼  PositionalEncoding(128)  [正弦或可学习]
  │
  ▼  4× EncoderBlock (Pre-LN)
  │     ├─ LayerNorm → MultiHeadAttention(8 heads, d_k=16) → residual + dropout
  │     └─ LayerNorm → FFN(128→512→128, GELU) → residual + dropout
  │
  ▼  LayerNorm
  │
  ▼  AttentionPooling(128)  [学习式注意力池化]
  │
  ▼  Linear(128→64) → GELU → Dropout → Linear(64→5)
  │
  ▼  Output (batch, 5)  — 未来 5 天预测收益率
```

总参数量约 50 万，适合 A 股有限数据量训练。

## 11. 风控机制

### 11.1 止损/止盈

| 规则 | 默认值 | 触发条件 |
|------|--------|----------|
| 固定止损 | -8% | (当前价 - 成本价) / 成本价 ≤ -8% |
| 止盈 | +15% | (当前价 - 成本价) / 成本价 ≥ +15% |
| 追踪止损 | 5% | 当前价 < 持仓期间最高价 × 95% |

三个条件按顺序检查，命中任一立即触发卖出。

### 11.2 仓位限制

- 最多同时持有 5 只股票
- 单只股票不超过组合净值 25%

### 11.3 回撤控制

| 阶段 | 回撤 | 行为 |
|------|------|------|
| 正常 | < 10% | 不干预 |
| 减仓 | 10% ~ 15% | 新买入信号强度 × 缩减因子 |
| 熔断 | ≥ 15% | 禁止所有新买入 |

缩减因子 = 1 - (回撤 - 10%) / 5%，线性递减。

## 12. 常见问题

### Q: 下载全 A 股需要多长时间？

约 5000 只股票，每只 0.5 秒间隔，行情 + 估值 + 财报共约 1.5 秒/只，总计约 2 小时。建议用 `--skip-cached` 断点续传。

### Q: 下载被限流怎么办？

脚本已内置 0.5 秒间隔。如仍被限流，可在代码中增大 `time.sleep()` 的值。失败的股票记录在 `outputs/logs/failed_*.txt`，可单独重跑。

### Q: 方向准确率只有 52%，正常吗？

正常。A 股市场中 52-55% 的方向准确率配合合理的风控即可盈利。IC > 0.03 属于可接受水平。

### Q: 如何修改预测天数？

修改 `config/default.yaml` 中的 `model.pred_len`，例如改为 3 天预测：

```yaml
model:
  pred_len: 3
```

### Q: 如何更换数据源为 tushare？

```yaml
data:
  source: "tushare"
  tushare_token: "你的token"
```

### Q: 模型检查点文件可以删除吗？

可以。`_best.pt` 是最佳模型，其他带时间戳的是训练中间检查点。删除不影响已训练好的模型加载。

### Q: 如何调整风控参数？

修改 `config/default.yaml` 中的 `risk` 部分。例如放宽止损到 10%：

```yaml
risk:
  stop_loss_pct: 0.10
```

改后重新运行回测即可验证效果。
